# Builder-Language Eval — v4: Cross-Domain Comprehension

Run timestamp: 2026-05-04T12:44:19.708Z

## What this measures

First eval to test the broad claim that motivated the skill: **across many declarative tasks across different domains, builder is a more compact and uniform grammar than markdown or prose.**

**Setup:**
- 5 tasks across 5 domains: project spec, intent over a collection, agent definition, CI/CD pipeline, multi-criteria audit.
- 3 formats per task: `builder` (with lambdas allowed for intent expressions), `markdown` (headers + bullets), `prose` (natural English paragraph).
- Hand-written for each format, equal effort, no reverse-translation.
- Two metrics:
  - **Compression** — chars to declare the same content
  - **Comprehension** — when a model reads the declaration and is asked to produce a manifest of every concrete element implied, what % of gold elements appear in the manifest?
- Comprehension run on Sonnet 4.6 + GPT-5.5 for cross-model robustness.
